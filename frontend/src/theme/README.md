# Theme System

The **AITrade Theme System** provides a centralized, type-safe design language for the entire application. It ensures visual consistency, maintainability, and scalability across all UI components.

## Overview

### Core Concepts

1. **Design Tokens** (`tokens.ts`): The single source of truth for all design values
   - Colors (surfaces, borders, text, brand, semantic)
   - Typography (font family, sizes, weights, line heights)
   - Spacing (8px base scale)
   - Border radius (xs → xl, full)
   - Shadows (sm, md, lg, glow)
   - Motion (transitions & easing)
   - Layout (viewport, grid, z-index)

2. **Theme Provider** (`ThemeProvider.tsx`): React Context for accessing tokens
   - Exposes design system to the component tree
   - `useTheme()` hook for functional components
   - `withTheme()` HOC for class components

3. **CSS Variables** (`cssVariables.ts`): Bridges TS tokens to CSS
   - Maps design tokens to CSS custom properties
   - Synchronized with `index.css` @theme block

4. **Utilities** (`utils.ts`): Helper functions for common patterns
   - Color selection by status
   - Semantic color variants
   - Contrast calculation
   - Transition effects

## Design Principle: Dark Theme

The application uses an **exclusive dark theme** with a professional color palette:

### Color Architecture

#### Surfaces (Hierarchy & Depth)

- **Page Background** (`#070c18`): Darkest — establishes the void
- **Card Surface** (`#0f1626`): Clearly distinguished from page
- **Input/Form Surface** (`#0b1220`): Slightly darker than card — sunken appearance
- **Hover/Active States** (`#16213a`, `#1a2845`): Progressive lightening for interactivity

#### Borders & Dividers

- **Primary Border** (`#1e3050`): Solid, visible separators
- **Light Border** (`#162440`): Subtle dividers, disabled states
- **Focus Ring** (`#6366f1`): Brand indigo for keyboard focus

#### Text

- **Primary** (`#e2e8f4`): Main content — high contrast for readability
- **Muted** (`#8894b0`): Secondary text — lower emphasis
- **Dim** (`#4a5878`): Tertiary — very subtle labels
- **Disabled** (`#3d4a62`): Non-interactive states

#### Brand (Primary Actions)

- **Indigo** (`#6366f1`): Action buttons, highlights, focal points
- **Hover** (`#818cf8`): Lighter on interaction
- **Dark** (`#4f46e5`): Pressed state
- **Glow Effects**: Subtle rgba overlays for depth

#### Semantic Status

- **Success** (`#10b981`): New data, positive actions
- **Danger** (`#f43f5e`): Errors, destructive actions
- **Warning** (`#f59e0b`): Alerts, caution states
- **Info** (`#06b6d4`): General information, neutral updates

Each semantic color includes base, glow, and light variants for flexible usage.

## Usage

### 1. Access Tokens in Components

```tsx
import { useTheme } from "@/theme";

export const MyComponent = () => {
  const { tokens } = useTheme();

  return (
    <div
      style={{
        backgroundColor: tokens.colors.surfaces.card,
        color: tokens.colors.text.primary,
        padding: tokens.spacing.md,
      }}
    >
      Content
    </div>
  );
};
```

### 2. Use CSS Variables (Preferred in Tailwind)

```tsx
export const Card = () => {
  return (
    <div className="bg-[var(--bg-card)] text-[var(--text)] rounded-[var(--radius-lg)] border border-[var(--border)]">
      Card content with theme colors
    </div>
  );
};
```

### 3. Apply Semantic Colors

```tsx
import { getColorByStatus, getSemanticColor } from "@/theme";

export const StatusBadge = ({
  status,
}: {
  status: "success" | "danger" | "warning" | "info";
}) => {
  const color = getColorByStatus(status);
  const { base, glow } = getSemanticColor(status);

  return (
    <span
      style={{
        backgroundColor: base,
        boxShadow: `0 0 12px ${glow}`,
      }}
    >
      {status.toUpperCase()}
    </span>
  );
};
```

### 4. Use Transition Utilities

```tsx
import { getTransition } from "@/theme";

export const InteractiveButton = () => {
  return (
    <button
      style={{
        transition: getTransition("all", "fast"),
        cursor: "pointer",
      }}
    >
      Click me
    </button>
  );
};
```

## Synchronization

The design system is defined in **two places**:

1. **TypeScript** (`tokens.ts`): The canonical source
2. **CSS** (`index.css` @theme block): Applied to the DOM

When modifying tokens:

- Update `tokens.ts` first
- CSS variables are automatically generated by `cssVariables.ts`
- Optionally call `injectThemeVariables()` at runtime

## Extending the Theme

### Adding a New Color

1. Add to `THEME_TOKENS.colors` in `tokens.ts`:

   ```ts
   export const THEME_TOKENS = {
     colors: {
       brand: {
         primary: "#6366f1",
         secondary: "#NEW_COLOR", // ← Add here
       },
     },
   };
   ```

2. Use in components:
   ```tsx
   const { tokens } = useTheme();
   const color = tokens.colors.brand.secondary;
   ```

### Adding New Motion/Animation

1. Add to `THEME_TOKENS.motion`:

   ```ts
   motion: {
     fast: '0.15s ease',
     verySlow: '0.5s ease-in-out', // ← Add here
   }
   ```

2. Use with the helper:
   ```ts
   const transition = getTransition("opacity", "verySlow");
   ```

## Best Practices

1. **Always use theme tokens** — Never hardcode colors
2. **Prefer CSS variables** — Better performance than JS object lookups
3. **Use semantic colors** — `success`, `danger`, `warning`, `info` for status
4. **Maintain consistency** — Apply padding/margins from `tokens.spacing`
5. **Test contrast** — Ensure text meets WCAG AA standards
6. **Document custom colors** — If adding non-token colors, explain why

- **UI**: Cards, Buttons, Inputs, Modals, Notifications
